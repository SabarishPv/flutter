Portable Flutter + Android SSD Setup

Your complete guide to creating and using a self-contained development environment on an external SSD for macOS.

Your Setup: Quick Reference

This section summarizes the complete folder structure and environment for your portable Flutter development setup. Use this as a reference for locating files and understanding the configuration.

SDK & Tools Directory Structure

/Volumes/TAP/app/
├── flutter/                # Flutter SDK
│   └── bin/                # Flutter and Dart executables
├── Android Studio.app      # Installed Android Studio
├── cmdline-tools/          # Android SDK command-line tools
│   └── latest/
│       └── bin/            # sdkmanager, avdmanager, etc.
├── platform-tools/         # adb, fastboot, etc.
├── platforms/
│   └── android-36/         # Android SDK platform, contains android.jar
├── build-tools/            # Android build tools
├── emulator/               # Android emulator executables
├── licenses/               # Accepted Android licenses
└── system-images/          # Android system images for emulators

Example Flutter Project Structure

/Volumes/TAP/app/FlutterProjects/todo_app/
├── lib/main.dart           # Entry point of the app
├── android/                # Android-specific files (Gradle, etc.)
├── pubspec.yaml            # Project dependencies and config
└── ...                     # Other project files

Environment Variable Configuration

# Placed in your ~/.zshrc file
export FLUTTER_HOME="/Volumes/TAP/app/flutter"
export ANDROID_SDK_ROOT="/Volumes/TAP/app"
export PATH="$FLUTTER_HOME/bin:$ANDROID_SDK_ROOT/emulator:$ANDROID_SDK_ROOT/platform-tools:$ANDROID_SDK_ROOT/cmdline-tools/latest/bin:$PATH"

Step 1: Prerequisites & Initial Setup

Before you begin, ensure you have the following ready in your main SSD directory (e.g., /Volumes/TAP/app/):

  • The Flutter SDK zip file downloaded and extracted into a folder named flutter.
  • Android Studio downloaded and placed as Android Studio.app.

This guide assumes your main development directory on the SSD is /Volumes/TAP/app/. All Android SDK components will be installed directly inside this folder.

Step 2: Install Android Command-Line Tools

The sdkmanager is essential for downloading SDK components. We need to download and place it correctly.

  1. Create the required directory structure:
mkdir -p /Volumes/TAP/app/cmdline-tools/latest
  1. Download the latest tools from the official Android Studio page. Then, run these commands to extract and place them. Remember to update the URL.
# Replace with the latest URL from the Android developer website
CMDLINE_TOOLS_URL="https://dl.google.com/android/repository/commandlinetools-mac-11076708_latest.zip"

curl -o ~/Downloads/cmdline-tools.zip "$CMDLINE_TOOLS_URL"
unzip -q ~/Downloads/cmdline-tools.zip -d ~/Downloads/
mv ~/Downloads/cmdline-tools/* /Volumes/TAP/app/cmdline-tools/latest/
rm ~/Downloads/cmdline-tools.zip
rm -rf ~/Downloads/cmdline-tools/

Step 3: Configure Your Shell Environment

Next, you need to tell your Mac's terminal where to find Flutter and the Android SDK tools by editing your .zshrc file.

# Open your .zshrc file in a text editor
open ~/.zshrc

# Add the following lines to the end of the file, then save it.
# This tells your system where to find the Flutter and Android tools.
export FLUTTER_HOME="/Volumes/TAP/app/flutter"
export ANDROID_SDK_ROOT="/Volumes/TAP/app"
export PATH="$FLUTTER_HOME/bin:$ANDROID_SDK_ROOT/emulator:$ANDROID_SDK_ROOT/platform-tools:$ANDROID_SDK_ROOT/cmdline-tools/latest/bin:$PATH"

Apply the changes by restarting your terminal or running:

source ~/.zshrc

Step 4: Install Android SDK Components

With sdkmanager configured, you can now install the necessary platforms, build tools, and the emulator.

sdkmanager --sdk_root=${ANDROID_SDK_ROOT} "platform-tools" "platforms;android-36" "build-tools;36.0.0" "emulator"

After installation, you must accept all licenses.

sdkmanager --licenses

You will be prompted several times. Press y and Enter to accept all licenses.

Step 5: Configure Flutter & Final Verification

Tell Flutter where to find the Android SDK and then run flutter doctor to confirm everything is working.

flutter config --android-sdk ${ANDROID_SDK_ROOT}

Now, run the final check:

flutter doctor -v

You should see green checkmarks for Flutter, Android toolchain, and Android Studio. It's okay to see errors for Xcode/iOS, as we are only targeting Android!

Step 6: Create and Run Your First App

Your environment is ready! Let's build something.

  1. Create a projects directory on your SSD and navigate into it.
mkdir -p /Volumes/TAP/app/FlutterProjects && cd /Volumes/TAP/app/FlutterProjects
  1. Use the Flutter CLI to create a new app.
flutter create my_first_ssd_app
  1. Run the app. Make sure an Android emulator is running or a physical device is connected via USB. 
cd my_first_ssd_app
flutter run
  1. To see an instant change, open lib/main.dart in your favorite code editor (like VS Code) and replace its contents with this simple "Hello World" code.
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        backgroundColor: Colors.teal[700],
        appBar: AppBar(
          title: const Text('My First SSD App'),
          backgroundColor: Colors.teal[900],
        ),
        body: const Center(
          child: Text(
            'Hello from my SSD!',
            style: TextStyle(
              fontSize: 24,
              color: Colors.white,
              fontWeight: FontWeight.bold,
            ),
          ),
        ),
      ),
    );
  }
}

After saving the file, Flutter's Hot Reload feature will automatically update the running application in seconds. Congratulations, you're now a Flutter developer!
